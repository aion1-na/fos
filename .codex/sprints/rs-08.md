RS-08 - Upgrade vectorized causal-ish transitions into an evidence-bounded transition engine with explicit uncertainty, target-trial estimands, and a hard guard against Concordia-created causal effects.

Objective: Upgrade vectorized causal-ish transitions into an evidence-bounded transition engine with explicit uncertainty, target-trial estimands, and a hard guard against Concordia-created causal effects.

Implement deliverables:
- TargetTrialSpec model and scenario compiler
- Transition kernel that separates measurement, causal transitions, network effects, Concordia cognition traces, and graph visualization artifacts
- Posterior draw and seed orchestration for uncertainty intervals
- Sensitivity modules: prior sensitivity, omitted-confounding/E-value hooks, seed stability, exposure-measure disagreement, and graph-layer ablation
- Model cards for each domain transition: happiness, health, meaning, character, relationships, and financial stability

Acceptance gates:
- Transition engine fails closed without required evidence priors and dataset_references
- Concordia and graph outputs cannot directly set causal effect sizes
- Every result includes expected effect, uncertainty intervals, probability of harm, subgroup heterogeneity, and data lineage
- A simple statistical baseline can run on the same run manifest for comparison

Constraints:
- Do not fabricate data. Fixtures are allowed only for unit tests and must be named fixture_only.
- Every production-facing dataset must have a dataset card, license/access metadata, codebook mapping, quality profile, provenance manifest, and content hash.
- Every simulation-facing output must use dataset_reference = (canonical_dataset_name, version, content_hash).
- Concordia outputs are qualitative interaction/cognition artifacts unless explicitly validated; Concordia must not create causal effect sizes.
- FOS Graph outputs are visual/analytic artifacts; graph layout or animation must not be treated as evidence.
- Keep changes scoped to this sprint and do not commit automatically.
