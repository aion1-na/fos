RS-05 - Add real community, time-use, health, and social-capital context needed for flourishing-domain transitions and network initialization.

Objective: Add real community, time-use, health, and social-capital context needed for flourishing-domain transitions and network initialization.

Implement deliverables:
- Connectors for GSS, ATUS, BRFSS, CDC WONDER where public, Pew Religious Landscape, Volunteering/Civic Life, Social Capital Atlas, and Opportunity Atlas
- Public-context connectors for World Happiness Report, ESS, WVS, OECD, World Bank, Eurostat, and ILO where needed
- features.community_context, features.time_use_context, features.health_validation_context, and features.social_capital_context
- Small-cell suppression policy checks for mortality outputs
- Dataset cards with limitations and inappropriate-use warnings

Acceptance gates:
- Community features can be joined to valid geography levels without leaking restricted detail
- Mortality/health tables respect public-use cell suppression rules
- Religious/community participation variables are modeled as voluntary social pathways, not prescriptive recommendations
- All context features carry content hashes and codebook versions

Constraints:
- Do not fabricate data. Fixtures are allowed only for unit tests and must be named fixture_only.
- Every production-facing dataset must have a dataset card, license/access metadata, codebook mapping, quality profile, provenance manifest, and content hash.
- Every simulation-facing output must use dataset_reference = (canonical_dataset_name, version, content_hash).
- MiroFish outputs are interaction/narrative artifacts unless explicitly validated; they must not create causal effect sizes.
- Keep changes scoped to this sprint and do not commit automatically.
