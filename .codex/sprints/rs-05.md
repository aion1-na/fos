RS-05 - Build the occupation, wage, labor-market, and AI-exposure substrate.

Objective: Build the real occupation, wage, labor-market, and AI-exposure substrate for the flagship AI/work/flourishing simulation.

Implement deliverables:
- Connectors for O*NET, BLS OEWS, BLS LAUS, BLS QCEW, CPS labor context, and BLS Employment Projections
- Public-archive ingesters for Eloundou GPT exposure, Felten AIOE, Webb exposure, Acemoglu/Restrepo robot exposure, and Anthropic Economic Index public releases where available
- Versioned SOC/O*NET/Census occupation crosswalks with preserved source labels
- features.occupation_ai_exposure_ensemble and features.us_young_adult_ai_work_context
- Exposure-disagreement uncertainty metadata and data cards

Acceptance gates:
- No headline exposure result uses a single exposure score
- Occupation crosswalks are versioned and preserve source codes
- Exposure ensemble reports measure disagreement by occupation and demographic subgroup
- Feature tables join to the RS-03 young-adult synthetic population and produce valid manifests

Constraints:
- Do not fabricate data. Fixtures are allowed only for unit tests and must be named fixture_only.
- Every production-facing dataset must have a dataset card, license/access metadata, codebook mapping, quality profile, provenance manifest, and content hash.
- Every simulation-facing output must use dataset_reference = (canonical_dataset_name, version, content_hash).
- Concordia outputs are qualitative interaction/cognition artifacts unless explicitly validated; Concordia must not create causal effect sizes.
- FOS Graph outputs are visual/analytic artifacts; graph layout or animation must not be treated as evidence.
- Keep changes scoped to this sprint and do not commit automatically.
