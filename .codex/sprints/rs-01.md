RS-01 - Harden the research data contract so simulation runs consume only versioned, content-addressed, provenance-tracked real-data artifacts.

Objective: Harden the research data contract so simulation runs consume only versioned, content-addressed, provenance-tracked real-data artifacts.

Implement deliverables:
- Shared Pydantic/TypeScript DatasetReference schema and RunDataManifest schema
- Data-service endpoints: resolve, card, manifest, lineage, feature_table, evidence_claim
- Simulation runtime guard that rejects unversioned dataset reads
- Provenance manifest emission for every run and scenario branch
- Dataset-card linter and fixture-only detector in CI

Acceptance gates:
- No simulation smoke test can access raw files directly
- Old dataset versions remain resolvable after new versions are registered
- Run manifest lists every dataset_reference touched by population synthesis, transition models, validation, and MiroFish adapter
- Missing dataset_reference returns a structured failure rather than silently substituting random data

Constraints:
- Do not fabricate data. Fixtures are allowed only for unit tests and must be named fixture_only.
- Every production-facing dataset must have a dataset card, license/access metadata, codebook mapping, quality profile, provenance manifest, and content hash.
- Every simulation-facing output must use dataset_reference = (canonical_dataset_name, version, content_hash).
- MiroFish outputs are interaction/narrative artifacts unless explicitly validated; they must not create causal effect sizes.
- Keep changes scoped to this sprint and do not commit automatically.
