RS-04 - Rename the platform to FOS and add the adapter registry.

Objective: Rename the platform to FOS across docs/runtime surfaces and create the adapter registry that lets FOS support Concordia, graph renderers, and future tools without binding scientific validity to any one external engine.

Implement deliverables:
- Controlled rename sweep from previous platform name to FOS across docs, UI copy, generated artifact labels, environment variables, and exported brief templates
- packages/fos-adapters with AdapterRegistry, ToolTrustPolicy, AdapterCapability, ToolArtifactReference, and ToolExecutionManifest schemas
- RunDataManifest extension: tool_artifacts, graph_artifacts, qualitative_artifacts, and adapter_versions
- docs/integrations/tool_trust_policy.md defining allowed, quarantined, blocked, and research-only adapters
- CI tests that forbid MiroFish/core coupling and forbid any external tool from bypassing dataset_reference contracts

Acceptance gates:
- A repo-wide grep produces no user-facing legacy platform name except migration notes and historical source references
- Core simulation tests pass with zero external adapters installed
- AdapterRegistry can register Concordia, cosmos.gl renderer, Sigma/Cytoscape fallback, and quarantined MiroFish reference without importing them
- Run manifests can record tool artifacts without changing prior RS-00 through RS-03 dataset contracts

Constraints:
- Do not fabricate data. Fixtures are allowed only for unit tests and must be named fixture_only.
- Every production-facing dataset must have a dataset card, license/access metadata, codebook mapping, quality profile, provenance manifest, and content hash.
- Every simulation-facing output must use dataset_reference = (canonical_dataset_name, version, content_hash).
- Concordia outputs are qualitative interaction/cognition artifacts unless explicitly validated; Concordia must not create causal effect sizes.
- FOS Graph outputs are visual/analytic artifacts; graph layout or animation must not be treated as evidence.
- Keep changes scoped to this sprint and do not commit automatically.
