# Contracts v0.1

This package defines the typed seam between simulation kernel code and domain packs.

`packages/contracts` intentionally contains no domain-specific contract fields. Pack-specific state belongs in `DomainPack.state_schema` and in concrete pack state models passed to `AgentState[T]`.

## Version

`CONTRACTS_VERSION` is `0.1.0`.

Domain packs declare `contracts_version`; loaders should call `assert_contracts_version(pack.contracts_version)` before using a pack.

## Appendix A Status

`docs/scope/v3.md` was not present when this version was implemented. The fields below are therefore minimal inferred definitions for v0.1 and should be reconciled against Appendix A when that source document is added.

## Minimal Inferred Fields

- `DomainPack`: pack identity, declared contracts version, opaque `state_schema`, ontology references, transition models, validation suites, render hints.
- `Scenario`: scenario identity, pack identity, display name, `stage_status`, ontology references, interventions, branches, shocks, opaque parameters.
- `AgentState[T]`: agent identity, simulation step, pack-supplied state model `T`, opaque metadata.
- `Population`: population identity, scenario identity, size, agent ids, opaque metadata.
- `PopulationSnapshot[T]`: snapshot identity, population identity, step, generic agent states, creation time.
- `SpawnSpec`: target population, count, opaque seed state, opaque metadata.
- `RunDataManifest`: run, scenario, population, dataset references, touched components, branch links, manifest hash, adapter versions, and tool/graph/qualitative artifact references.
- `ToolArtifactReference`: adapter artifact id, adapter id, artifact type, URI, content hash, required dataset references, causal-validation flag, and notes.
- `SimulationRun`: run identity, scenario identity, population identity, lifecycle status, timestamps, optional fidelity report, opaque outputs.
- `EvidenceClaim`: claim identity, optional scenario identity, statement, source URI, confidence, opaque metadata.
- `ValidationReport`: report identity, run identity, suite identity, status, evidence claims, opaque metrics, errors.
- `OntologyRef`: ontology id, optional version, optional URI.
- `TransitionModel`: transition id, version, entrypoint, opaque parameters schema.
- `Intervention`: intervention id, label, opaque parameters, optional step bounds.
- `ValidationSuite`: suite id, check ids, opaque parameters.
- `RenderHints`: preferred view ids and opaque encoding hints.
- `BranchSpec`: branch id, label, optional parent id, opaque parameters.
- `ShockSpec`: shock id, label, target step, opaque parameters.
- `FidelityReport`: fidelity level, notes, opaque metrics.

## Generated Artifacts

JSON Schema files are generated under `packages/contracts/schemas/v0.1/`.

TypeScript artifacts are generated with:

```bash
uv run python tools/codegen/contracts_to_ts.py
```

Generated TypeScript output is written to `packages/contracts/dist/ts/` and exported by `@fos/contracts`.
