# Scientific Review Checklist

Use this checklist before any demo, research-grade, or decision-grade output is shared outside the project team.

- Scenario definition is explicit and versioned.
- Dataset references are content-addressed with canonical dataset name, version, and content hash.
- Validation gates are listed with pass, warning, or fail status.
- Seed policy is documented, including seed count and deterministic replay requirements.
- Dataset version policy is documented, including old-version resolvability.
- Construct, codebook, and evidence-claim sign-off status is recorded.
- Limitations and inappropriate uses are visible in the public or private brief.
- Demo language does not imply validated causal or decision-grade claims.
